# v1.0 Release Gate

Do not tag `v1.0.0` until every gate below is satisfied.

## Repository And CI

- `v1.0.0` tag and `main` are identical, or release notes explain the difference.
- Tag CI and main CI pass on Python 3.10 and 3.11.
- CI runs pytest, trigger smoke, prompt eval report freshness, adapter eval status, image-output record validation, and good/bad/warn strict lint.

## Prompt Eval

- Every prompt case has `source_prompt`, `rewritten_prompt_candidate`, and `skill_output_prompt`.
- Every `skill_output_prompt` has `skill_output_source`, `skill_output_date`, and `skill_output_notes`.
- `evals/run_prompt_eval.py` shows Source, Candidate, Skill, Skill Delta, Skill Source, and Missing Features.
- Missing Features must be empty or explicitly justified before release.

## Image-Output Eval

At least five real scored records must exist in `evals/image_output_records.json`:

- `gpt_image_text_rendering`
- `gpt_image_edit_preserve_change`
- `midjourney_no_phrase`
- `flux_plain_negation`
- `series_identity_drift`

Each scored record must include:

- `model_version`
- `adapter_version`
- `prompt_before`
- `rewritten_prompt`
- `skill_output_source`
- `skill_output_date`
- `skill_output_notes`
- `task_type`
- `generation_params`
- `output_image_path`
- `image_score`
- `task_gate_results`
- `human_rater`
- `observed_failures`
- `revision_prompt`
- `final_score`

Run:

```bash
python evals/check_image_output_records.py --require-real-records
```

The script must pass. Output paths must exist locally or use stable `http(s)` URLs. Task gates must pass. `image_score` values must be `0`, `1`, `2`, or `"not_applicable"`. `final_score` must equal the sum of numeric `image_score` values and meet the rubric threshold.

`observed_failures` may be empty for an individual record only when no visible failure was found, but v1.0 must include at least two records with non-empty `observed_failures` and concrete `revision_prompt` values.

## Adapter Evidence

- Any model with scored image-output records must update `Image-output eval last run` in `model-adapters.md`.
- Models without real output records must remain marked as prompt-level or heuristic only.
- Do not imply output-quality validation for a model that only has prompt-level lint fixtures.

## Documentation

- README must distinguish prompt-level validation from image-output validation.
- README must link the fixed tag CI run, prompt eval report, image-output records, and rubric.
- Known limitations must continue to say the linter is not an image-quality guarantee.
